1. Basic 版本的问题

  • ✗ 性能反而下降，尤其在 GAIA 上（−26.91）
  • 原因：任务过于简单，模型学会了"抄近道"
  • 后果：过拟合简单模式，泛化能力变差
  • 教训：仅仅增加实体密度还不够，任务复杂度同样重要

2. Union 版本的提升

  • ✓ 平均提升 +3.26，大部分基准都有提升
  • 原因：多源信息整合迫使模型进行复杂推理
  • 意义：证明了结构复杂度的重要性

3. Reverse-Union 版本的最佳效果

  • ✓ 平均提升 +4.34，在大多数基准上最好
  • 原因：
    • 不仅结构复杂，还有逆向推理设计
    • 杜绝了关键词搜索捷径
    • 先演绎 → 再规划 → 再搜索（锻炼高级认知能力）
  • 特别优势：在 WideSearch 上提升最大（+10.92）
结论：任务复杂度的递进设计至关重要。Reverse-Union 通过逆向推理，从根本上提升了代理的规划和推理能力。
